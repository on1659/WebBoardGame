## 프롬프트

7세 아이를 위한 웹 기반 체스 게임을 React로 만들어줘.

### 기술 스택
- React (Vite)
- chess.js 라이브러리 (체스 규칙/유효성 검증)
- AI: Minimax 알고리즘 + Alpha-Beta Pruning (chess.js 기반)
- 스타일: CSS Modules 또는 styled-components (취향에 맞게)
- 추가 라이브러리는 최소화

### 핵심 기능

**1. 체스 보드 & 기물**
- 8x8 체스 보드 렌더링
- 기물은 유니코드 체스 심볼 사용 (♔♕♖♗♘♙♚♛♜♝♞♟) — 크고 선명하게
- 드래그 앤 드롭 + 클릭 이동 둘 다 지원
- 플레이어는 항상 백(아래), AI는 흑(위)

**2. AI 상대 (Minimax)**
- chess.js로 합법 수 생성 → Minimax + Alpha-Beta Pruning으로 최선의 수 선택
- 난이도에 따라 탐색 깊이 조절:
  - 쉬움: depth 1 (+ 랜덤성 추가해서 일부러 실수하게)
  - 보통: depth 2
  - 어려움: depth 3~4
- 평가 함수: 기물 가치 + 위치 보너스 테이블(PST)
- AI가 수를 두는 동안 "생각 중..." 표시 (setTimeout으로 0.5~1초 딜레이)

**3. 난이도 선택**
- 게임 시작 전 난이도 선택 화면
- 쉬움 / 보통 / 어려움 3단계
- 각 난이도에 귀여운 아이콘이나 이모지 표시 (🐣 🐱 🦁 등)
- 게임 중에도 설정에서 변경 가능

**4. 이동 가능 칸 하이라이트 (토글 옵션)**
- 기물을 선택하면 이동 가능한 칸을 시각적으로 표시
- 빈 칸: 반투명 원형 도트
- 적 기물이 있는 칸: 테두리 하이라이트
- 설정에서 ON/OFF 토글 가능 (기본값: ON)

**5. 게임 상태 표시**
- 현재 턴 표시 (네 차례! / AI가 생각 중...)
- 체크 상태일 때 킹 칸 빨간색 하이라이트 + "체크!" 알림
- 체크메이트/스테일메이트/무승부 감지 → 결과 모달 표시
- 잡힌 기물 목록 표시 (양쪽 각각)

**6. 추가 기능**
- 무르기 (Undo) 버튼: 마지막 수 취소 (플레이어+AI 수 함께 되돌림)
- 새 게임 버튼
- 수 기록 표시 (대수기보법, 사이드바에 스크롤 가능)

### UI/디자인 가이드

**콘셉트: 귀여운 동물 숲 느낌**
- 부드러운 파스텔 톤 배경 (연한 민트, 라벤더, 피치 등)
- 보드 색상: 밝은 나무색 + 연한 녹색 (눈이 편한 조합)
- 둥근 모서리(border-radius) 적극 활용
- 부드러운 그림자(box-shadow)로 카드 느낌
- 기물 선택/이동 시 부드러운 애니메이션 (CSS transition)
- 버튼에 호버 애니메이션
- 폰트: 둥글둥글하고 친근한 느낌 (Google Fonts에서 'Jua' 또는 'Gaegu' 같은 한글 폰트)
- 체크메이트 시 축하 애니메이션 (confetti 또는 별 이펙트)
- 전체적으로 아이가 보기에 무섭지 않고 친근한 분위기

**레이아웃**
- 중앙에 체스 보드
- 좌측 또는 상단: 게임 정보 (턴, 잡힌 기물)
- 우측 또는 하단: 수 기록, 설정 패널

### 반응형 & 터치 디바이스 (iPad / iPhone 필수 지원)

**보드 크기 적응**
- 보드는 항상 `min(vw, vh)`에 맞게 정사각형 유지
- iPhone SE(375px) ~ iPad Pro(1024px)까지 대응
- 데스크톱: 보드 + 사이드 패널 가로 배치
- 태블릿(768px 이하): 보드 위/아래로 정보 패널 세로 배치
- 모바일(480px 이하): 보드가 화면 폭의 95% 차지, 나머지 UI는 아래로

**터치 조작 (핵심)**
- 드래그 앤 드롭은 터치에서 불안정할 수 있으므로, **탭-탭 방식을 기본**으로:
  1. 기물 탭 → 선택됨 (하이라이트)
  2. 이동할 칸 탭 → 이동 실행
  3. 빈 칸이나 같은 기물 탭 → 선택 해제
- 드래그도 지원하되 터치 이벤트(touchstart/touchmove/touchend) 별도 처리
- **터치 타겟 최소 44x44px** (Apple HIG 기준)
- 기물이 손가락에 가려지지 않도록 드래그 시 기물을 손가락 위로 오프셋

**iOS/Safari 대응**
- `-webkit-touch-callout: none` (길게 누르기 팝업 방지)
- `-webkit-user-select: none` (텍스트 선택 방지)
- `touch-action: manipulation` (더블탭 줌 방지)
- `overflow: hidden` on body (바운스 스크롤 방지)
- `position: fixed` 레이아웃 대신 `dvh`(dynamic viewport height) 사용
- 100vh 대신 `100dvh` 사용 (Safari 주소창 때문에 높이 차이 남)
- 보드 영역에서 스크롤/줌 제스처 차단 (`touch-action: none`)

**7세 아이 터치 UX**
- 칸 크기가 최소 40px 이상 유지 (작은 화면에서도)
- 선택된 기물에 확실한 시각 피드백 (배경색 변경 + 살짝 확대 애니메이션)
- 잘못 터치해도 쉽게 취소 가능 (다른 곳 탭하면 해제)
- 버튼들도 충분히 크게 (최소 높이 48px, 간격 12px 이상)
- 실수 방지: 유효하지 않은 이동은 무시 + 부드러운 흔들림 애니메이션으로 피드백

### 주의사항
- chess.js의 moves(), move(), undo(), isCheckmate(), isStalemate(), isDraw(), isCheck() 등 활용
- AI 계산은 Web Worker에 넣으면 좋지만, 깊이가 낮으므로 메인 스레드에서 setTimeout으로도 충분
- 프로모션(폰이 끝까지 가면): 7세 아이는 자동으로 퀸 승격이 편함 (옵션으로 선택 가능하면 더 좋음)
- 한글 UI: 모든 텍스트는 한글로 표시
